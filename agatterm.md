Erstellen Sie eine Java Anwendung, welche nach dem Model-View-Controller Prinzip arbeitet – siehe https://www.tutorialspoint.com/design_pattern/mvc_pattern.htm Spielregeln Der Computer generiert zu Beginn einen geheimen Code aus '4 Farben'. Folgende Farben stehen zur Verfügung: 'R, G, B, Y, O, P' (Rot, Grün, Blau, Gelb, Orange, Pink). Farben dürfen sich 'wiederholen'. Der Spieler hat '10 Versuche' den Code zu erraten. Ein Versuch besteht aus der Eingabe von 4 Farbbuchstaben, z. B. R G B Y. Der Computer gibt nach jedem Versuch eine Auswertung in Form von Treffern zurück: '●' = richtige Farbe an der richtigen Position '○' = richtige Farbe, aber falsche Position Wird der Code vollständig richtig erraten, gewinnt der Spieler. Werden alle 10 Versuche verbraucht, ohne dass der Code erraten wurde, endet das Spiel unentschieden. Der geheime Code wird anschließend angezeigt. Anforderungen Die Anwendung muss strikt nach dem Model-View-Controller-Prinzip aufgebaut sein. Das 'Model' enthält sämtliche Spiellogik: Erzeugen des geheimen Codes Überprüfen der Benutzereingabe Auswertung der Versuche Verwaltung der verbleibenden Versuche Die 'View' ist für Darstellung und Benutzereingaben verantwortlich: Anzeige der bisherigen Versuche und deren Auswertung (versuchen Sie eine möglichst ähnliche Darstellung wie im Originalspiel) Ausgabe der verbleibenden Versuche Einlesen eines gültigen Farbcodes (4 Buchstaben, erlaubte Farben) Der 'Controller' steuert den gesamten Spielablauf und verbindet Model und View. Die Eingabe des Spielers muss solange wiederholt werden, bis ein gültiger Code eingegeben wurde (korrekte Länge, erlaubte Zeichen). Nach jedem Versuch muss überprüft werden, ob: der Spieler gewonnen hat (alle 4 Positionen korrekt) oder das Spiel unentschieden ist (alle Versuche verbraucht) Am Ende soll eine entsprechende Gewinn- oder Verlustmeldung ausgegeben werden.
